# 采购合同管理

## 需求概述
基于已批准的采购订单生成正式采购合同，支持 AI 自动生成合同文本、审批流转、合同编号回填、Word/PDF 导出。

## 功能清单
1. **执行单位管理** — 在系统设置中管理合同甲方（执行单位）信息，含名称、地址、银行账号、统一社会信用代码等
2. **合同列表** — 项目内合同列表，支持按流程状态筛选、按合同编号/名称/供应商搜索
3. **新建合同** — 3 步骤流程：选择采购订单（仅同供应商已批准订单）→ 填写合同变量字段（税率、付款比例等下拉选择）→ AI 生成预览 → 提交
4. **合同详情** — 展示合同基本信息、关联订单、物资明细、合同正文，支持审批操作（提交/批准/驳回）
5. **合同编辑** — 仅草稿/已驳回状态可编辑变量字段，支持重新生成合同内容
6. **审批流程** — 草稿 → 审批中 → 已批准/已驳回
7. **合同编号回填** — 审批通过时自动回填合同编号到所有关联采购订单的 contractNo 字段
8. **删除清理** — 草稿合同删除时清空关联订单的 contractNo
9. **AI 合同生成** — 基于内置模板（10 章 14 条）+ 变量值 + 物资明细 + 双方信息，使用 OpenAI 兼容客户端（支持 DeepSeek V4）生成结构化合同文本
10. **Word/PDF 导出** — Word 使用 docx 库直接下载 .docx；PDF 通过浏览器打印另存 PDF（轻量方案）
11. **采购订单联动** — 采购订单列表显示"生成合同"按钮（已批准且无合同的订单）+ 显示已签合同编号

## 注意事项
1. **流程状态规范**：使用全局 4 状态（草稿/审批中/已批准/已驳回），与其他模块统一
2. **硬删除**：所有删除操作走 DELETE 接口直接删除数据库记录，符合硬删除规范
3. **变量字段全部下拉选择**：避免手输错误，所有合同变量（税率/付款比例/质保期/交货期限/运输费用/违约金）都是下拉选项
4. **AI 模型**：默认通过 OpenAI 兼容客户端调用，DeepSeek V4 等其他模型只需在 AI 模型配置中设置 base URL 即可使用
5. **合同内容存储**：AI 生成的合同正文存为 JSON（ProcurementContract.content），详情页实时渲染，不存完整 HTML
6. **回填逻辑**：审批通过时遍历所有关联订单回填 contractNo，删除时清空，保证数据一致性
7. **合同模板**：硬编码在 `src/lib/contract-template.ts`，不作为可管理的动态数据
8. **金额字段**：当前用 Float 类型（与项目现有保持一致），未来如需精确计算建议统一迁移到 Decimal
9. **导出方案差异**：Word 是真实文件下载；PDF 是 HTML 打印方案，让用户用浏览器"另存为 PDF"
