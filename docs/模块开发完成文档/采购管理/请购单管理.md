# 请购单管理

## 需求概述
实现请购单的完整生命周期管理，包括创建、审批、采购跟踪到自动关闭的全流程，支持多明细行操作与双状态管理。

## 功能清单

### 基础功能
1. **请购单字段** - 编号（自动生成 `REQ-YYYYMMDD-XXX`）、申请人、采购分类、需求类型、专业、期望到货日期、备注
2. **请购单明细** - 物料名称、规格、材质、牌号、标准规范、数量、单位、用途、需求日期
3. **明细行操作** - 支持添加、删除、编辑数量
4. **已采购数量跟踪** - 自动计算已订购量与剩余量，实时展示

### 双状态管理
5. **流程状态**（列表页下拉选择） - 草稿 → 已批准
   - 控制编辑/删除权限：仅"草稿"状态可编辑和删除
   - 后续对接正式审批流程时，审批节点对应此状态
6. **业务状态**（系统自动计算，只读展示） - 正常 → 部分采购 → 关闭
   - 创建采购订单后自动变为"部分采购"
   - 所有明细行剩余量为0时自动变为"关闭"
   - 删除所有关联订单后恢复为"正常"
7. **采购订单引用** - 已批准且业务状态为"正常"或"部分采购"的请购单可被采购订单引用

### Excel导入
8. **Excel导入物料** - 支持 .xlsx/.xls 格式，客户端直接解析

## 注意事项
- 双状态设计：流程状态管权限，业务状态管采购进度，互不干扰
- 创建采购订单并关联请购单后，请购单的**业务状态**会自动变为"部分采购"或"关闭"
- 编号规则：`REQ-` + 日期（YYYYMMDD）+ 3 位序号（如 REQ-20260605-001）
- 流程状态当前为手动下拉选择，后续对接审批流后会自动流转
- 明细行删除时需确认是否已关联采购订单，已订购数量不为 0 的行不可删除
